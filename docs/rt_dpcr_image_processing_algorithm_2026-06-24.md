# Real-Time dPCR Image Processing Algorithm

Date: 2026-06-24

This note records the current image-processing and curve-classification workflow used for the four RT-dPCR concentration experiments. The current implementation is in:

`draw_new_method_nature_figures.py`

The main output adds a 10^-4-specific uncertain overlay while keeping the original mode-aware kinetic classification results for the other concentrations.

## Goal

The algorithm processes real-time fluorescence images from microwell-array dPCR experiments and converts them into per-well amplification curves, endpoint maps, positive/negative calls, and quality-control visualizations.

The key requirement of the current version is:

1. Keep the original positive/negative classification for 10^-2, 10^-3, and 10^-5.
2. In the 10^-4 mixed-concentration experiment, mark the negative/edge curves that overlap the positive amplification cluster as uncertain.
3. Display those 10^-4 uncertain wells in gray and exclude them from the adjusted 10^-4 quantitative count.

## Processing Flow

### 1. Read Per-Well Curves

The script reads both positive and negative curve CSV files for each experiment. Each curve point contains:

- well coordinates: `x`, `y`
- cycle index
- image filename
- raw fluorescence intensity
- smoothed plotted fluorescence value

The well key is generated as:

`xy_key = round(x) + "_" + round(y)`

This key is used to merge curve data with kinetic classification features.

### 2. Merge Kinetic Classification Features

For every well, the script merges:

- `classification_after`
- `classification_before`
- `is_rescued`
- `is_rejected`
- `is_uncertain`
- `decision_status`
- `cq`
- `kinetic_score`

`classification_after` is renamed to `final_call` for plotting.

### 3. Read Curve Outlier Flags

The workflow outlier table is merged by `xy_key`. The display-level outlier flag is built from available columns:

- `is_hidden`
- `is_curve_outlier`
- `is_early_cycle_outlier`
- `is_late_drop_outlier`

Outlier curves are shown in pale yellow in the curve plots.

### 4. Apply Experiment-Level Vertical Offset

For visualization, curves are shifted using the upper quantile of early-cycle fluorescence:

```text
offset = 97.5% quantile of early-cycle smoothed fluorescence
display_curve = smoothed_plotted_value - offset
```

This keeps raw curve shapes intact while making cross-experiment curve panels visually comparable.

### 5. Standard Positive/Negative Curve Plot

The original figure uses:

- red: `final_call == positive`
- blue: `final_call == negative`
- pale yellow: plot-level rejected/outlier curves

This original output is still generated:

`nature_new_method_positive_negative_curves.png`

### 6. 10^-4 Uncertain Overlay

The new figure keeps the original plot style, but modifies only the 10^-4 panel.

For 10^-4:

- wells with `is_uncertain == True` are removed from the red/blue positive-negative display groups
- these wells are plotted in gray as `uncertain`
- positive and negative counts in the 10^-4 panel are recalculated after excluding the uncertain wells

This preserves the visual meaning of the curve clusters:

- red: confident positive
- blue: confident negative
- gray: mixed/uncertain kinetic behavior
- pale yellow: rejected or plot-level outlier

The uncertain group corresponds to 10^-4 wells whose curve dynamics overlap with the positive amplification region even though their endpoint/decision status is not confidently positive.

## Interpretation

The 10^-4 experiment sits in a mixed occupancy region. Some curves classified as negative or boundary wells show amplification kinetics close to the positive cluster. Marking them as uncertain avoids treating these ambiguous partitions as confident negatives. This makes the curve figure more consistent with the actual kinetic behavior and gives a clearer quality-control narrative for real-time dPCR.
